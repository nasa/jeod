82 /**
83  * BodyAction is the base class for the BodyAction model.
84  * A BodyAction instance that performs some operation on a MassBody object.
85  * The simulation Dynamics Manager object manages a collection of BodyAction
86  * objects for the purpose of initializing MassBody objects and later, for
87  * performing asynchronous actions on them.
88  *
89  * The BodyAction model hinges on three methods:
90  * - initialize()
91  * The initialize() method initializes the BodyAction.
92  * This method does not and must not operate on the subject of the action.
93  * All derived classes must forward the initialize() call to the immediate
94  * parent class and then perform class-dependent object initializations.
95  * - is_ready()
96  * The is_ready method indicates whether the action is ready to be applied.
97  * For example, an action that initializes the translation state of a
98  * vehicle relative to some other vehicle cannot do its job until that
99  * other vehicle's translational state is set. The is_ready() method for such
100  * an action should return false until the other vehicle's translational
101  * state has been set.
102  * - apply()
103  * The apply() method applies the action -- it does something to the subject
104  * of the action.
105  * All derived classes must perform class-dependent actions and then
106  * must forward the apply() call to the immediate parent class.
107  */
