81 /**
82  * BodyAction is the base class for the BodyAction model.
83  * A BodyAction instance that performs some operation on a MassBody object.
84  * The simulation Dynamics Manager object manages a collection of BodyAction
85  * objects for the purpose of initializing MassBody objects and later, for
86  * performing asynchronous actions on them.
87  *
88  * The BodyAction model hinges on three methods:
89  * - initialize()
90  * The initialize() method initializes the BodyAction.
91  * This method does not and must not operate on the subject of the action.
92  * All derived classes must forward the initialize() call to the immediate
93  * parent class and then perform class-dependent object initializations.
94  * - is_ready()
95  * The is_ready method indicates whether the action is ready to be applied.
96  * For example, an action that initializes the translation state of a
97  * vehicle relative to some other vehicle cannot do its job until that
98  * other vehicle's translational state is set. The is_ready() method for such
99  * an action should return false until the other vehicle's translational
100  * state has been set.
101  * - apply()
102  * The apply() method applies the action -- it does something to the subject
103  * of the action.
104  * All derived classes must perform class-dependent actions and then
105  * must forward the apply() call to the immediate parent class.
106  */
