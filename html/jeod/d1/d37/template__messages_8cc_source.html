14  * The MessageHandler defines several methods for informing the user of
15  * special conditions that can arise, ranging in severity from fatal errors to
16  * detailed debug information. Each of these methods takes a message_code
17  * argument. The intent of the message code is to identify the model that
18  * invoked the message handler and to characterize the nature of the message.
19  *
20  * JEOD models implement that intent by making each of the message codes be a
21  * static data member of a model-specific message class. Each such data member
22  * points to a literal character string of the form "model/path/message_id",
23  * where "model/path" is the path to the model, and "message_id" is the
24  * stringified data member name.

52  The goal in a model message code implementation file is to create storage for
53  and initialize each of static data members declared in the model message class.
54  There are several ways to do this. The examples below target the fictitious
55  message class FooMessages defined in the model utils/foo. The class defines
56  two message codes, FooMessages::fatal and FooMessages::warning.
57  - Most primitive:@verbatim
58  const char * FooMessages::fatal = "utils/foo/fatal";
59  const char * FooMessages::warning = "utils/foo/warning";
60  @endverbatim
61  - Recognizing that "utils/foo/" is common:@verbatim
62  #define PATH "utils/foo/"
63  const char * FooMessages::fatal = PATH "fatal";
64  const char * FooMessages::warning = PATH "warning";
65  @endverbatim
66  - Using the JEOD_MAKE_MESSAGE_CODE macro:@verbatim
67  #define PATH "utils/foo/"
68  JEOD_MAKE_MESSAGE_CODE (FooMessages, PATH, fatal);
69  JEOD_MAKE_MESSAGE_CODE (FooMessages, PATH, warning);
70  @endverbatim
71  - Using the model-specific macro MAKE_FOO_MESSAGE_CODE:@verbatim
72  #define MAKE_FOO_MESSAGE_CODE(id) \
73  JEOD_MAKE_MESSAGE_CODE(FooMessages, "utils/foo/", id)
74  MAKE_FOO_MESSAGE_CODE (fatal);
75  MAKE_FOO_MESSAGE_CODE (warning);
76  @endverbatim
