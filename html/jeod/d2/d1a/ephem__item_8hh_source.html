82 /**
83  * The EphemerisItem class is the base class for representing an item that is
84  * modeled in an ephemeris model. Ephemeris items form the bridge between the
85  * reference frame model and the ephemeris models.
86  * An EphemerisItem
87  * - Has a name, which is not necessarily unique. Ephemeris items with the same
88  * name are linked to one another to form a linked list.
89  * - Has a target reference frame. This is the ephemeris reference frame which
90  * has the same name as the ephemeris item.
91  * - Can be enabled or disabled. At most one item of a set of commonly-named
92  * items can be enabled, and only if a correspondingly-named ephemeris
93  * reference frame exists. Ownership of the correspondingly-named ephemeris
94  * reference frame transfers to the enabled ephemeris item. When an ephemeris
95  * item is disabled, the ephemeris model that owns the ephemeris item should
96  * not operate on the correspondingly-named reference frame.
97  * - Can be active or inactive. Disabled items must always be inactive. The
98  * activity level of the enabled item for a set of commonly-named items is
99  * automatically maintained to be the same as that of the target frame.
100  *
101  * The ephemeris model that owns an active ephemeris item is responsible
102  * for ensuring that the correspondingly-named ephemeris reference frame is
103  * a part of the active reference frame tree and for updating that reference
104  * frame's state.
105  */
