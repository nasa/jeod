112 /**
113  * A Trick::IntegLoopScheduler that provides the ability to integrate a
114  * collection of Trick::SimObject instances over time, with the sim objects
115  * capable of being moved from one integration loop to another during run time.
116  * A JeodDynbodyIntegrationLoop augments this capability in a number of regards:
117  * - All DynBody objects contained in the sim objects integrated by a
118  * JeodDynbodyIntegrationLoop object are integrated using JEOD integration.
119  * - The DynBody objects to be integrated by a JeodDynbodyIntegrationLoop object
120  * are automatically collected as a member of the DynamicsIntegrationGroup
121  * object contained within a JeodDynbodyIntegrationLoop object.
122  * - Non-DynBody integrable objects can also be integrated using
123  * JEOD integration.
124  * - Non-DynBody integrable objects that are elsewhere identified as being
125  * associated with a DynBody object are automatically collected along with
126  * the DynBody objects with which they are associated.
127  * - The DynBody and associated integrable objects are integrated using the
128  * DynamicsIntegrationGroup object contained in the loop object.
129  *
130  * Users of this class are strongly encouraged to do so via a
131  * JeodIntegLoopSimObject.
132  * See $JEOD_HOME/lib/jeod/JEOD_S_modules/integ_loop.sm.
133  */
