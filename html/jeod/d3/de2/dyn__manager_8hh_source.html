94 /**
95  * The DynManager class manages the dynamic elements of a simulation.
96  * The primary functions of a DynManager are to:
97  * - Dynamically determine which ephemerides are needed in a simulation.
98  * - Initialize ephemeris models and keep them in sync with the rest
99  * of the simulation.
100  * - Initialize mass bodies and dynamic bodies independently of the order
101  * in which these bodies are declared in the S_define file.
102  * - Coordinate the computation of the cumulative forces and torques and
103  * gravitational effects on the dynamic bodies in a simulation.
104  * - Coordinate the integration of time and and of dynamic body states.
105  * - Apply asynchronous actions to bodies.
106  *
107  * The DynManager can operate in one of three modes: empty space,
108  * single planet, and ephemeris mode. The DynManager inherits from
109  * EphemerisInterface so that when it operates in empty space or
110  * single-planet mode it can properly register itself as
111  * the owner of the reference frame tree root node.
112  */

114  virtual public BaseDynManager,
