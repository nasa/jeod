116 /**
117  * Denote that error messages whose message code exactly matches the supplied
118  * code is a result of an condition being tested in some unit test. The
119  * registered 'allowed codes' are used two cases: fatal errors and non-fatal
120  * but severe errors, the latter defined in terms of the fatal threshold.
121  *
122  * The default exit status for program termination resulting from a call to
123  * MessageHandle::fail is one. When the message code matches one of the
124  * allowed codes the exit status will be zero. Since the JEOD unit test driver
125  * only looks at the program termination status, the allowed code mechanism
126  * provides a means for failure into success. What this means to you as a
127  * unit test developer is that you can now test the error tests in the mainline
128  * code. It also means you need to be careful in how you use this capability;
129  * maybe that successful failure is not as successful as you thought.
130  *
131  * Two things happen when a non-fatal but sufficiently serious MessageHandler
132  * call is made whose the message code matches one of the allowed codes:
133  * - The retrievable allowed_error_count is bumped. This provides a means
134  * for detecting in your unit test code that the an error condition was
135  * properly detected.
136  * - The test for a verboten code is bypassed. The allowed codes trump the
137  * verboten codes in case of conflict.
138  * \param[in] code The message code to be marked as allowed
139  */
140 void TestSimInterface::add_allowed_code(const std::string & code)
141 {
142  message_handler.add_allowed_code(code);
143 }

145 /**
146  * Denote that an error message whose message code partially matches the
147  * supplied code is bad news even if the error is not a fatal error.
148  * The test is of the first N characters of the incoming message code versus
149  * the verboten code, where N is the string length of the incoming code.
150  * The empty string as a verboten code will match all messages.
151  *
152  * The test will be terminated with non-zero exit status when an incoming
153  * non-fatal message
154  * - Has a severity level less than or equal to the failure_threshold,
155  * - Has a message code that does not match (exact match) any of the
156  * allowed codes, and
157  * - Has a message code that does match (initial match) one of the
158  * verboten codes.
159  * \param[in] code The message code to be marked as verboten
160  */
161 void TestSimInterface::add_verboten_code(const std::string & code)
162 {
163  message_handler.add_verboten_code(code);
164 }
