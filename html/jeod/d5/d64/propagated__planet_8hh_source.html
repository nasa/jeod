87  * A PropagatedEphemerisPlanet is an EphemerisPoint whose state
88  * is coupled with the translational state of a DynBody reference frame.
89  *
90  * This class is intended for use by the PropagatedPlanet class. Use outside of
91  * the PropagatedPlanet is not sanctioned.
92  *
93  * The inherited enabled flag takes on an additional meaning in this derived
94  * class. The base class meaning of this flag is that the ephemeris item, or the
95  * ephemeris model that owns the item, is responsible for maintaining the
96  * translational state of the target frame when the item is enabled but not when
97  * the item is disabled.
98  *
99  * For this class, when the enabled flag is set still means that the item is
100  * responsible for maintaining the translational state of the target ephemeris
101  * reference frame. The dynamic body serves as the source of this state.
102  * When the enabled flag is clear, some other model is assumed to be responsible
103  * for maintaining the target frame state. The target frame state serves as the
104  * source of the dynamic body's state when the enabled flag is clear.
105  */

185 /**
186  * The PropagatedPlanet ephemeris model provides planetary state via a DynBody
187  * object whose state is propagated using the JEOD state integration techniques.
188  * Scenarios in which a simulation will use a PropagatedPlanet object include:
189  * - An object such as an asteroid for which an ephemeris model is not readily
190  * available.
191  * - An object such as a planet that is represented in some other ephemeris
192  * model but the simulation developer wants the planet to be propagated to
193  * ensure that the planet and the vehicles operating in the vicinity of the
194  * planet obey the same laws of physics.
195  *
196  * The PropagatedPlanet model provides mechanisms that accommodate these
197  * scenarios. The class defines multiple modes in which a propagated planet
198  * planet object operates. In all modes, the model ensures consistency between
199  * the translational states of the dynamic body's composite frame and the
200  * planet's planet-centered frame and between the rotational states of the
201  * dynamic body's composite frame and the planet's planet-fixed frame.
202  */
203 class PropagatedPlanet : public EphemerisInterface
204 {
205  JEOD_MAKE_SIM_INTERFACES(jeod, PropagatedPlanet)
206 
207 public:
208  // Types
209 
210  /**
211  * Defines the modes in which an active PropagatedPlanet object operates.
212  * A PropagatedPlanet contains a BasePlanet pointer PropagatedPlanet::planet
213  * and a DynBody PropagatedPlanet::body. The translational states of the
214  * planet-centered inertial frame and the body's composite_body frame are
215  * tied to one another, as are the rotational states of the planet-fixed
216  * frame and the the body's composite_body frame. This enum identifies
217  * which of the planet or the body is the source of translational and the
218  * the rotational parts of the state.
219  */
220  enum Mode
221  {
222  TransFromPlanet_RotFromPlanet = 0, /*
223  The planet-centered inertial frame and planet-fixed frame are the
224  sources of the translational and rotational states. The dynamic body's
225  composite frame is slaved to these planetary sources. */
226  TransFromPlanet_RotFromBody = 1, /*
227  The planet-centered inertial frame is the source of the translational
228  state and the dynamic body's composite frame is the source of the
229  rotational state. The composite frame's translational state and the
230  planet-fixed frame's rotational state are slaved to these sources. */
231  TransFromBody_RotFromPlanet = 2, /*
232  The dynamic body's composite frame is the source of the translational
233  state and the planet-fixed frame is the source of the rotational state.
234  The planet-centered inertial translational state and the composite
235  frame's rotational state are slaved to these sources. */
236  TransFromBody_RotFromBody = 3 /*
237  The dynamic body's composite frame is the source of the translational
238  and rotational states. The planet-centered inertial frame and the
239  planet-fixed frame are slaved to these dynamic body sources. */
240  };
