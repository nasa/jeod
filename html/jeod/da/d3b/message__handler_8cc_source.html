62  * Generate a message with negative severity, MessageHandler::Failure,
63  * and terminate the simulation.
64  * The intent of this method is to handle erroneous situations for which
65  * no recovery path exists. If a recovery path does exist, even if very
66  * suspect, callers of this method should consider calling
67  * MessageHandler::error as an alternative.
68  *
69  * A conforming implementation of a class that derives MessageHandler
70  * will force the simulation to terminate upon receipt of a
71  * message with negative severity.

73  * \param[in] line Typically __LINE__

509 /**
510  * Handle the error condition where there is no global handler.
511  * @note That this condition exists means the simulation is non-compliant.
512  *
513  * \par Assumptions and Limitations
514  * - All JEOD-based simulations must have a message handler and memory handler
515  * that at instantiated prior to instantiating any other JEOD-based model
516  * and destroyed after all those other models have been destroyed.
517  * - That no message handler exists means the simulation is not a compliant
