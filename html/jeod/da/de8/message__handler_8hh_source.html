79 /**
80  * The base class for generating JEOD messages.
81  * This class provides:
82  * - A suite of public static message generation and message control functions.
83  * The message generation functions provide the mechanism to generate and
84  * present messages of various levels of severity to the simulation user.
85  * The message control functions provide the user the ability to control
86  * which messages will be presented and to control their presentation.
87  * - A set of defined constants that denote the severity levels used by JEOD.
88  * - Failure (-1): Any negative severity level value indicates an
89  * irrecoverable error. The simulation will terminate immediately.
90  * - Error (0): Errors almost certainly invalidate the simulation output.
91  * All severity zero messages are printed; they cannot be disabled.
92  * The difference between failures and errors is that there is some
93  * recovery from an error that lets the simulation limp on and find the
94  * next error.
95  * - Warning (9): Warnings represent conditions that a model deems to be
96  * suspect but not necessarily dangerous. Warning messages are printed
97  * by default, but they can be disabled.
98  * - Notice (99): Notices represent condiitions that a model deems to be
99  * suspicious, but not necessarily in error. Notifications are not
100  * printed by default, but they can be enabled.
101  * - Debug (999): Debug messages typically demonstrate progress of some
102  * sort. Enabling them may well result in spew.
103  * - A public default constructor and destructor.
104  * The constructor ensures that the created object is indeed a singleton.
105  * -
106  */
