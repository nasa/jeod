80 /**
81  * The base class for generating JEOD messages.
82  * This class provides:
83  * - A suite of public static message generation and message control functions.
84  * The message generation functions provide the mechanism to generate and
85  * present messages of various levels of severity to the simulation user.
86  * The message control functions provide the user the ability to control
87  * which messages will be presented and to control their presentation.
88  * - A set of defined constants that denote the severity levels used by JEOD.
89  * - Failure (-1): Any negative severity level value indicates an
90  * irrecoverable error. The simulation will terminate immediately.
91  * - Error (0): Errors almost certainly invalidate the simulation output.
92  * All severity zero messages are printed; they cannot be disabled.
93  * The difference between failures and errors is that there is some
94  * recovery from an error that lets the simulation limp on and find the
95  * next error.
96  * - Warning (9): Warnings represent conditions that a model deems to be
97  * suspect but not necessarily dangerous. Warning messages are printed
98  * by default, but they can be disabled.
99  * - Notice (99): Notices represent condiitions that a model deems to be
100  * suspicious, but not necessarily in error. Notifications are not
101  * printed by default, but they can be enabled.
102  * - Debug (999): Debug messages typically demonstrate progress of some
103  * sort. Enabling them may well result in spew.
104  * - A public default constructor and destructor.
105  * The constructor ensures that the created object is indeed a singleton.
106  * -
107  */
