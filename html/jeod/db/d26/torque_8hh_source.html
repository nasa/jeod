67 /**
68  * A Torque represents a Newtonian torque that acts on a DynBody.
69  * The class encapsulates an active flag and a 3-vector that contains the
70  * torque components. Torques are collected in one of a DynBody object's
71  * torque collection STL vectors. The torque vector is expressed in the
72  * structural frame of that DynBody object.
73  *
74  * The Torque class is the recommended mechanism for representing torques in
75  * JEOD. While 3-vectors can also be collected into a collect STL vector,
76  * theee is is no way to turn off these collected 3-vectors. Even worse, there
77  * is no way to tell whether a collected 3-vector does indeed represent a
78  * torque, or even if it is a 3-vector. In comparison, Torque objects can be
79  * turned on and off, and more importantly, they are type-safe.
80  */
81 class Torque
82 {
83 public:
84  // Member functions
85  Torque() = default;
86  virtual ~Torque() = default;
87  Torque(const Torque &) = delete;
88  Torque & operator=(const Torque &) = delete;

93  * Is this torque active?
94  */

96  /**
97  * Torque vector

108 /**
109  * A CollectTorque represents a collected torque that acts on a vehicle.
110  * The BodyTorqueCollect class contains STL vectors that in turn contain
111  * CollectTorque pointers. These vectors are populated via the Trick vcollect
112  * mechanism. A Trick simulation issues vcollect statements such as
113  * \code
114  * vcollect vehicle.body.collect.collect_XXX_forc CollectTorque::create {
115  * vehicle.torque_model1.torque,
116  * vehicle.torque_model2.torque
117  * };
118  * \endcode
119  * This invokes the appropriate CollectTorque create method on each listed
120  * element.
121  *
122  * CollectTorques should not be used in model code to represent torques.
123  * Use the Torque class instead.
